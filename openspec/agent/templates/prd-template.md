# 任务：[功能名称]

**输入**：来自用户需求的 `[###-功能名称]/` 的设计文档
**组织方式**：任务按用户故事分组，以支持每个用户故事的独立实现和测试。

## 用户故事1：`[ID] [P?] [故事] 描述`

- **[P]**：可以并行运行（不同文件，无依赖）
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径

## 依赖的服务端接口

- **[ID]**：服务端接口的 ID
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径

## 依赖的前端接口

- **[ID]**：前端接口的 ID
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径

## 依赖的前端组件

- **[ID]**：前端组件的 ID
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径

## 依赖的前端页面

- **[ID]**：前端页面的 ID
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径

## 依赖的埋点信息

- **[ID]**：埋点信息的 ID
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径
