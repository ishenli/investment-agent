# 任务：[功能名称]

**输入**：来自 `/specs/[###-功能名称]/` 的设计文档
**前置条件**：plan.md（必需），spec.md（用户故事必需），research.md，data-model.md
**参考**：[SmallFish 框架规范](file://.codefuserules/SmallFish%E6%A1%86%E6%9E%B6%E6%8A%80%E6%9C%AF%E8%A7%84%E8%8C%83.md)

**测试**：
- 代码检查：`npm run lint`

**组织方式**：任务按用户故事分组，以支持每个故事的独立实现和测试。

## 格式：`[ID] [P?] [故事] 描述`

- **[P]**：可以并行运行（不同文件，无依赖）
- **[故事]**：此任务所属的用户故事（例如：US1, US2, US3）
- 在描述中包含确切的文件路径

## 路径约定

- **页面**：`src/pages/[页面名称]/`
- **服务**：`src/services/`
- **存储**：`src/stores/`
- **组件**：`src/components/`
- **钩子**：`src/hooks/`
- **样式**：`src/styles/`

## 依赖关系与执行顺序

### 阶段依赖关系

- **设置（第1阶段）**：可以立即开始
- **基础（第2阶段）**：依赖设置 - 阻塞UI工作
- **用户故事（第3阶段+）**：依赖基础阶段
- **完善**：依赖所有期望的用户故事完成

### 每个用户故事内

- 类型/服务/存储通常在UI之前
- 子组件在主页面组装之前
- 静态UI在交互集成之前

### 并行机会

- 不同的子组件可以并行构建
- 服务/存储实现可以与类型定义并行运行
